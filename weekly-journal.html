Week 1
January 8-14, 2025
Research & Discovery

Project Initiation & Problem Understanding

Key Progress:

  • Conducted initial research on bat conservation and light pollution in Glasgow
  • Identified GlasGLOW festival as a key case study for temporary lighting impacts
  • Established team roles and communication protocols
  • Created project charter and initial timeline

Key Decisions:

  • Focus Area: Chose Glasgow parks as primary monitoring locations
  • Target Users: Defined primary stakeholders as city council and conservation groups
  • Technology Approach: Decided on web-based dashboard for accessibility

Challenges & Solutions:

Challenge: Limited access to existing bat monitoring data
Solution: Reached out to local wildlife organizations and research institutions
Challenge: Team members in different time zones
Solution: Established regular meeting schedule accommodating all members
Week 2
January 15-21, 2025
Stakeholder Research

Stakeholder Interviews & Requirements Gathering

Key Progress:

  • Conducted interviews with 3 Glasgow City Council environmental officers
  • Spoke with 2 local bat conservation experts
  • Analyzed existing lighting policies and bat protection guidelines
  • Created initial user personas and journey maps

Key Decisions:

  • Data Sources: Prioritized real-time sensor data over periodic surveys
  • User Interface: Decided on dashboard approach over mobile app
  • Community Feature: Added adopt-a-bat program based on stakeholder feedback

Challenges & Solutions:

Challenge: Stakeholders had conflicting priorities for features
Solution: Created feature priority matrix based on impact vs. effort
Week 3
January 22-28, 2025
Ideation & Concept Development

Brainstorming & Initial Concept Design

Key Progress:

  • Conducted 2 collaborative brainstorming sessions
  • Generated 15+ initial concept ideas
  • Created first paper sketches and wireframes
  • Developed initial information architecture

Key Decisions:

  • Core Features: Finalized dashboard structure with 4 main sections
  • Visual Direction: Chose nature-inspired color palette with high contrast
  • Technical Stack: Selected HTML/CSS/JS with Chart.js for visualizations

Challenges & Solutions:

Challenge: Too many feature ideas without clear prioritization
Solution: Used MoSCoW method to categorize features into Must/Should/Could/Won't have
Week 4
January 29 - February 4, 2025
Lo-fi Prototyping

Paper Prototypes & User Flow Testing

Key Progress:

  • Created detailed paper prototypes for all major user flows
  • Tested navigation concepts with 5 potential users
  • Refined information architecture based on user feedback
  • Started digital wireframes in Figma

Key Decisions:

  • Navigation: Simplified main menu to 7 core sections
  • Dashboard Layout: Prioritized key metrics at top with detailed charts below
  • Mobile Strategy: Decided on responsive design rather than separate mobile app

Challenges & Solutions:

Challenge: Users confused by technical terminology
Solution: Added glossary and simplified language throughout interface
Week 5
February 5-11, 2025
Hi-fi Design

High-Fidelity Design & Visual System

Key Progress:

  • Completed high-fidelity designs for all 7 main pages
  • Established comprehensive design system with colors, typography, and components
  • Created interactive Figma prototype with transitions
  • Conducted accessibility review and made adjustments

Key Decisions:

  • Typography: Selected Inter font family for excellent readability
  • Color System: Implemented dark theme with blue/green accents
  • Charts: Chose Chart.js for interactive data visualizations

Challenges & Solutions:

Challenge: Charts difficult to read on mobile devices
Solution: Created simplified mobile chart views with option to expand

Week 6
February 12-18, 2025
Development

Frontend Development & Implementation

Key Progress:

  • Built responsive HTML/CSS framework for all pages
  • Implemented interactive dashboard with Chart.js
  • Added mobile navigation and responsive features
  • Created GitHub repository and set up version control

Key Decisions:

  • Hosting: Selected GitHub Pages for free static site hosting
  • Performance: Optimized for fast loading with minimal dependencies
  • Browser Support: Ensured compatibility with all modern browsers

Challenges & Solutions:

Challenge: Complex charts causing performance issues on mobile
Solution: Implemented progressive loading and simplified mobile views
Challenge: Cross-browser compatibility issues with CSS Grid
Solution: Added fallbacks using Flexbox for older browser support

Key Lessons Learned

User-Centered Design

Early and frequent user testing prevented major usability issues and ensured our solution met real user needs.

Iterative Development

Regular feedback loops and iterative improvements led to a much stronger final product than initial concepts.

Feature Prioritization

Learning to say no to good ideas that didn't serve core user needs was crucial for project success.

Mobile-First Approach

Starting with mobile constraints led to cleaner, more focused designs across all device sizes.
